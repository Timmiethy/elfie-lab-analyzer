Why it's different

An engine.

Most "AI health" demos = one LLM call in a wrapper. We built an 11-stage typed pipeline where the model is one tool, not the system.

The pipeline · architecture

Eleven typed stages. One contract per boundary.

Pink = deterministic core. Blue = rule-driven. Dotted = lineage tap on every stage.

Stage 1 · Ingestion

Two lanes. One contract.

Same Pydantic contract

Both lanes emit Observation(label, value, unit, span, source). Downstream doesn't care where it came from.

Failure isolation

VLM flake = lane skipped, not pipeline crash. PDF OCR miss = surfaced as support_state=unsupported, never hallucinated.

Source preserved

Every observation carries its OCR span. Lineage can replay the pixel back to the patient sentence.

Stage 3 · Rule-based normalization

One raw label → LOINC code. Zero inference.

Five deterministic layers. If all fail, we say unsupported — we do not guess.

The deterministic core

Same input = same output. Always.

The hard parts we solved
01

Heterogeneous inputs

PDF + image lanes. Mineru for structured PDFs. VLM for photographed reports.

02

Analyte identity

"WBC" = "white blood cell count" = "leukocytes". Alias table → token signature → LOINC.

03

Unit safety

mg/dL vs mmol/L = coma. UCUM validates and converts before any rule fires.

04

Rules as data

Severity, panels, next-steps live in versioned YAML policy packs. No deploy to update.

05

Lineage everywhere

Every field carries its provenance chain. No black box. Fully auditable.
